## 简介
本文件全面解析GeometryTD中技能经验值系统的设计与实现，涵盖经验值获取机制、等级提升系统、分配策略、可视化反馈以及游戏平衡设计。系统采用基于时间的自动经验获取，支持均匀分配、随机分配与指定槽位分配三种模式，并通过UI提供直观的升级反馈与粒子动画。

**更新** 系统现已采用Fisher-Yates洗牌算法优化随机分配机制，提供更加公平和数学上正确的随机化过程。

## 项目结构
经验值系统主要由以下模块构成：
- 技能管理器：负责经验值的累计、溢出处理与等级提升判定，包含优化的随机分配算法

- UI层：提供浮动文字提示与粒子特效展示
- 配置层：定义经验获取速率、范围与上限等参数
- 事件系统：支持通过事件触发经验授予

### Fisher-Yates洗牌算法优化的随机分配

**更新** 系统现已采用Fisher-Yates洗牌算法优化随机分配机制，提供更加公平和数学上正确的随机化过程。

#### GrantXpToSlots方法的优化随机分配算法
- 均匀分配：slotCount < 0时，对所有槽位均分经验
- 优化随机分配：slotCount ≥ 0时，使用Fisher-Yates洗牌算法对可升级槽位进行完全随机化，然后选择前count个槽位
- Fisher-Yates算法：从数组末尾开始，对每个位置i，随机选择一个范围在[0, i]内的索引j，然后交换elements[i]和elements[j]
- 安全循环：限制最大尝试次数(100)，避免极端情况下死循环

```mermaid
flowchart TD
Enter(["GrantXpToSlots(xpAmount, slotCount)"]) --> Empty{"slots为空?"}
Empty --> |是| Return["返回"]
Empty --> |否| Mode{"slotCount < 0 ?"}
Mode --> |是| All["遍历所有槽位<br/>AddXpToSlot(i, xpAmount)"]
Mode --> |否| Collect["收集可升级槽位索引"]
Collect --> Shuffle["Fisher-Yates洗牌算法<br/>for i = n-1 to 1<br/>j = Random(0..i)<br/>swap(elements[i], elements[j])"]
Shuffle --> Select["选择前count个槽位"]
Select --> Loop["遍历前count个槽位"]
Loop --> Add["AddXpToSlot(idx, xpAmount)"] --> Loop
All --> Return
Loop --> Done{"已处理完所有槽位?"}
Done --> |是| Return
```

### AddXpToSlot方法的经验值累加逻辑与溢出处理
- 累加：直接将amount加到当前槽位xp
- 溢出：当xp≥10且等级<10时，持续扣减10并提升一级，直至无法再升级
- 回调：每次成功升级触发OnSlotLevelUp事件，供UI层订阅


## 性能考量
- 时间驱动：经验获取在Update中按帧处理，开销极低
- 优化随机选择：PickRandomEligibleSlot与AddXpToRandomSlot均为O(n)扫描，n为槽位数，通常较小
- Fisher-Yates算法：GrantXpToSlots中的洗牌算法复杂度为O(k)，k为可升级槽位数量
- UI协程：FloatingTextUI与SkillXpTimerUI使用协程，生命周期受对象销毁影响，注意内存释放

**更新** Fisher-Yates洗牌算法虽然增加了少量计算开销，但提供了更好的随机性质量，对于通常较小的槽位数量来说性能影响微乎其微。

## 故障排除指南
- 经验不增长
  - 检查BattleManager的skillXpTimer是否正确递减
  - 确认SkillManager的AddXpToSlot未被提前返回（索引越界或已达10级）
- 升级无提示
  - 检查SkillBarUI是否正确订阅OnSlotLevelUp
  - 确认FloatingTextUI的Show调用路径与颜色参数
- 粒子特效不播放
  - 检查SkillXpTimerUI的PlayXpParticle目标RectTransform是否有效
  - 确认Canvas层级与排序设置
- 随机分配不公平
  - 检查Fisher-Yates洗牌算法是否正确执行
  - 确认可升级槽位列表是否正确收集

**更新** 新增了随机分配公平性的故障排除指导。
