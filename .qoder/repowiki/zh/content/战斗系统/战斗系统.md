## 简介
本文件系统化梳理 GeometryTD 的回合制战斗机制与实时战斗融合的实现，覆盖战斗状态管理、行动顺序控制、回合切换逻辑；基于 UnitController 基类的角色控制器（HeroController、MonsterController、BossController、SummonMonsterController）的职责与协作；敌人生成系统（MonsterSpawner）的波次管理与难度递增；伤害计算（包含命中、暴击、元素抗性与加成）与 Buff/被动影响；以及战斗结果判定与奖励分配。同时提供扩展性设计说明，指导如何新增敌人类型、Boss 挑战与特殊战斗场景。

**更新** 本次更新重点介绍了基于 UnitController 基类的统一架构重构，显著减少了代码重复并提高了可维护性。**新增** 角色定制系统（VisualLoader）为战斗单位提供了动态外观定制能力，支持基于 RoleConfig 的角色配置系统集成。**新增** GameHelper 动画加载增强支持 RuntimeAnimatorController 的动态加载，**新增** UnitController 视觉初始化集成实现了完整的角色外观加载流程。

## 项目结构
战斗系统主要位于 Assets/Scripts/Battle 目录下，围绕 BattleManager 作为中枢协调各子系统，现在采用基于 UnitController 的统一架构：
- BattleManager：战斗生命周期、生成与销毁、UI 更新、Boss 事件链、奖励发放、**视觉特效管理**
- **统一基类**：UnitController（抽象基类）提供所有战斗单位的通用行为逻辑，包括统一的标识系统
- **统一标识系统**：所有战斗单位现在都有 UID（唯一标识符）、UnitGroup（阵营）和 UnitType（类型）三个核心标识
- **角色定制系统**：VisualLoader 提供基于 RoleConfig 的动态外观定制，支持 Animator 和 Sprite 的动态替换
- 角色控制器：HeroController（玩家）、MonsterController（小怪）、BossController（Boss）、SummonMonsterController（召唤物）
- 生成器：MonsterSpawner（波次、精英、Boss 触发）
- 属性与效果：AttrComponent（属性基值/加成/派生）、BuffSystem（增益减益）、PassiveSystem（被动触发）
- 计算器：DamageCalculator（伤害、命中、暴击、元素抗性）
- 技能系统：SkillManager（技能槽、经验、冷却）
- **特效系统**：EventEffectManager（视觉特效管理）、AttachedEffectController（持续特效控制器）
- **子弹追踪系统**：BulletController（增强的 homing 追踪机制，支持弧线飞行、目标寻找和路径碰撞检测）

## 核心组件
- 战斗中枢 BattleManager：负责初始化、生成敌人与子弹、AOE 伤害、Boss 事件链、胜利/失败判定、UI 更新与奖励发放、**视觉特效管理**
- **统一基类 UnitController**：抽象基类，提供所有战斗单位的通用行为逻辑，包括属性管理、技能系统、Buff 管理、攻击逻辑、死亡处理等，**新增统一标识系统**
- **统一标识系统**：所有战斗单位现在都具备三个核心标识
  - UID（唯一标识符）：BattleManager.GenerateUid() 生成的唯一整数标识
  - UnitGroup（阵营）：UnitGroup 枚举，包括 Neutral（中立）、Player（玩家）、Enemy（敌方）
  - UnitType（类型）：UnitType 枚举，包括 Hero（英雄）、Monster（普通怪物）、Boss（Boss）、Summon（召唤物）
- **角色定制系统 VisualLoader**：基于 RoleConfig 的动态外观定制组件，支持 Animator 和 Sprite 的动态替换，提供资源兜底机制
- **角色配置系统 RoleConfig**：角色配置数据结构，包含角色ID、名称、预制体路径、动画控制器路径、槽位贴图配置等
- **GameHelper 动画加载增强**：支持 RuntimeAnimatorController 的动态加载，提供 Resources 和 Editor 环境下的双路径加载
- 英雄 HeroController：基于 UnitController 的具体实现，负责普攻与技能释放、蓄力状态、护盾、伤害吸收、伤害文本显示、事件执行
- 小怪 MonsterController：基于 UnitController 的具体实现，负责寻路、近战或远程攻击、死亡回收
- Boss BossController：基于 UnitController 的具体实现，负责移动到目标位置、定点攻击、Boss 血量 UI 更新
- **召唤物 SummonMonsterController**：基于 UnitController 的具体实现，负责限时存在、自动攻击、基于召唤者属性缩放
- 生成器 MonsterSpawner：负责按波次生成普通敌人、精英敌人、Boss 触发与通关判定
- 属性 AttrComponent：提供属性基值、加成、派生属性（如最大生命、攻击力、攻速、移速）
- 增益减益 BuffSystem：统一管理 Buff 生命周期、叠层、属性加成、跳伤、特殊效果（冻结、无敌、反击等）、**持续特效挂载**
- 被动 PassiveSystem：被动触发、条件判断、移除策略、**一次性特效触发**
- 伤害计算器 DamageCalculator：命中、闪避、元素加成/减免、暴击、Boss/精英加成
- 技能系统 SkillManager：技能槽、等级、经验、冷却、使用流程
- **视觉特效系统 EventEffectManager**：管理 event_effect_config.json 驱动的视觉特效，支持一次性特效和持续挂载特效
- **子弹追踪系统 BulletController**：增强的 homing 追踪机制，支持弧线飞行、目标寻找和路径碰撞检测

## 架构总览
战斗系统采用"中枢协调 + 统一基类 + 角色自治 + 视觉定制"的架构：
- BattleManager 作为全局中枢，统一调度生成、销毁、UI、事件与奖励，**集成视觉特效管理**
- **UnitController 作为统一基类，提供所有战斗单位的通用行为逻辑，减少代码重复并提高可维护性，新增统一标识系统**
- **VisualLoader 作为角色定制系统，集成到 UnitController 的视觉初始化流程中，提供基于 RoleConfig 的动态外观定制**
- 角色控制器各自维护状态机（移动、攻击、技能、死亡），通过 AttrComponent 与 BuffSystem 协作
- 生成器独立于战斗循环外，按固定间隔触发生成
- 计算器与系统以纯函数方式提供可复用的计算能力
- **特效系统独立于战斗逻辑，通过事件驱动的方式提供视觉反馈**

### 角色定制系统（VisualLoader）
**新增** VisualLoader 是一个专门用于动态加载角色视觉表现的组件，为战斗系统提供了完整的角色定制能力：

- **动态外观定制**：根据 RoleConfig 配置动态替换槽位 Sprite 和 Animator
- **资源兜底机制**：当资源不存在时自动隐藏槽位并打印警告日志
- **自动槽位收集**：支持通过节点命名规范自动识别和收集槽位
- **手动槽位注册**：支持动态生成场景的手动槽位注册
- **命名规范支持**：支持 `{slotType}_Slot` 和 `Slot_{slotType}` 两种命名格式
- **资源路径规范**：支持 Resources 和 Editor 环境下的双路径加载

### 统一标识系统详解
**新增** 所有战斗单位现在都具备统一的标识系统，这是新架构的重要组成部分：

- **UID（唯一标识符）**：通过 BattleManager.GenerateUid() 生成的唯一整数标识，用于区分不同的战斗单位实例
- **UnitGroup（阵营）**：UnitGroup 枚举，包括：
  - Neutral（中立）：中立单位，通常不参与战斗
  - Player（玩家）：玩家控制的单位，包括英雄和召唤物
  - Enemy（敌方）：敌方单位，包括普通怪物和 Boss
- **UnitType（类型）**：UnitType 枚举，包括：
  - Hero（英雄）：玩家英雄单位
  - Monster（普通怪物）：普通敌方单位
  - Boss（Boss）：强大的敌方 Boss 单位
  - Summon（召唤物）：由英雄或其他单位召唤的临时单位

这些标识通过 InitUnit() 方法在单位初始化时设置，确保每个单位都有完整的身份信息。

### 伤害计算系统（命中/闪避/元素/暴击）
伤害计算遵循以下步骤：
1) 命中判定：命中率 - 闪避率，若低于阈值则 Miss
2) 基础伤害：攻击 × 技能伤害比例 / 10000
3) 元素加成：攻击侧元素伤害加成 + 全元素伤害加成
4) 元素减免：防御侧对应元素伤害减免 + 全元素伤害减免
5) 暴击判定：攻击侧暴击率 - 防御侧暴击抵抗，产生暴击倍率（考虑暴击伤害抵抗）
6) Boss/精英加成：根据目标类型追加伤害加成
7) 最终伤害：取最大 0 的整数结果

## 性能考量
- **统一基类优化**：UnitController 减少了代码重复，避免了角色控制器之间的重复计算
- **统一标识系统优化**：通过 BattleManager.GenerateUid() 生成唯一标识符，避免了手动管理 UID 的复杂性
- **角色定制系统优化**：VisualLoader 采用字典映射和缓存机制，避免重复的节点查找和资源加载
- **资源加载优化**：GameHelper 支持 Resources 和 Editor 环境下的双路径加载，减少资源查找开销
- 更新频率：角色控制器每帧仅做必要计算（Buff Tick、冷却、寻路/攻击），避免高频 GC
- 数据结构：使用 List 存活敌人与 Buff，注意及时清理无效项
- 伤害计算：纯函数计算，避免频繁对象分配；命中/暴击使用整数比例减少浮点误差
- 生成策略：波次间隔固定，精英与 Boss 阈值基于整数计数，避免复杂判定
- **特效管理**：EventEffectManager 使用协程管理特效生命周期，避免阻塞主线程
- **追踪优化**：BulletController 使用向量距离计算和 HashSet 避免重复命中检测

## 故障排查指南
- **基类问题**：如果角色控制器出现通用问题，检查 UnitController 的实现
- **标识系统问题**：如果单位出现 UID 冲突或阵营/类型错误，检查 InitUnit() 方法的调用
- **角色定制问题**：如果 VisualLoader 无法正确加载外观，检查 RoleConfig 配置和资源路径
- **槽位识别问题**：如果槽位无法被正确识别，检查节点命名规范（`{slotType}_Slot` 或 `Slot_{slotType}`）
- **资源加载问题**：如果资源加载失败，检查 GameHelper 的资源路径和 Assets 结构
- 英雄无法释放技能：检查 SkillManager 的槽位等级与冷却剩余时间
- 命中异常：核对命中率/闪避率配置与 AttrComponent 的派生属性
- Buff 不生效：确认 BuffSystem 的叠加上限、持续时间与属性加成是否正确
- Boss 不出现：检查 MonsterSpawner 的 killCount 与 bossList 阈值
- 伤害为 0：检查元素加成/减免与 Boss/精英加成是否导致最终伤害被截断
- **特效不显示**：检查 event_effect_config.json 配置是否正确，特效预制件是否存在
- **追踪失效**：检查 BulletController 的 homing 标志和目标有效性
- **穿透异常**：确认 BulletEventData 的 pierceCount 配置和碰撞检测逻辑

### 新增组件详细说明

#### VisualLoader 组件
**新增** VisualLoader 是一个专门用于动态加载角色视觉表现的组件，具有以下特性：
- **动态外观定制**：根据 RoleConfig 配置动态替换槽位 Sprite 和 Animator
- **资源兜底机制**：当资源不存在时自动隐藏槽位并打印警告日志
- **自动槽位收集**：支持通过节点命名规范自动识别和收集槽位
- **手动槽位注册**：支持动态生成场景的手动槽位注册
- **命名规范支持**：支持 `{slotType}_Slot` 和 `Slot_{slotType}` 两种命名格式
- **资源路径规范**：支持 Resources 和 Editor 环境下的双路径加载

#### GameHelper 动画加载增强
**新增** GameHelper 的 LoadAnimator 方法增强了动画控制器的动态加载能力：
- **RuntimeAnimatorController 支持**：支持动态加载 RuntimeAnimatorController 资源
- **双路径加载**：优先尝试 Resources.Load，失败时在 Editor 环境下通过 AssetDatabase 加载
- **错误处理**：当资源不存在时打印警告日志并返回 null

#### RoleConfig 角色配置系统
**新增** RoleConfig 提供了完整的角色配置数据结构：
- **角色ID管理**：通过 id 字段唯一标识不同角色类型
- **预制体路径**：通过 prefabPath 指定基础预制体路径
- **动画控制器路径**：通过 animatorPath 指定自定义动画控制器路径
- **槽位贴图配置**：通过 sprite_set 数组配置各个槽位的贴图路径
- **头像配置**：通过 head 字段配置对话用头像路径

#### EffectBurstAnim 和 EffectFadeAnim 特效动画
**新增** 这两个特效动画组件提供了基础的视觉效果支持：
- **EffectBurstAnim**：快速缩放+淡出的爆发动画，支持外部初始化
- **EffectFadeAnim**：持久动画：大部分持续时间保持颜色，然后淡出后自我销毁
- **初始化支持**：都支持通过 Init() 方法进行外部初始化

#### UnitController 视觉初始化集成
**新增** UnitController 集成了 VisualLoader 的视觉初始化功能：
- **InitVisual 方法**：提供虚拟的 InitVisual(roleId) 方法，供子类重写
- **RoleConfig 集成**：通过 Cfg.Role.Get(roleId) 获取角色配置
- **VisualLoader 调用**：调用 visualLoader.LoadVisual(config) 完成外观加载
- **错误处理**：当找不到 VisualLoader 组件时打印警告日志

这些改进显著提升了战斗的策略性和视觉表现力，为玩家提供了更加丰富和流畅的游戏体验。角色定制系统的引入为游戏提供了更强的个性化能力，GameHelper 的增强为资源管理提供了更好的支持，为未来的扩展和维护奠定了坚实的基础。
